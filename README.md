# Skystore Project

## Описание

"Skystore" — это полнофункциональный интернет-магазин на Django, включающий в себя каталог товаров и интегрированный блог. Платформа реализует полный CRUD-функционал для продуктов и статей, кастомную модель пользователя с регистрацией и авторизацией, а также разграничение прав доступа. Проект демонстрирует современные практики веб-разработки, включая кастомную валидацию форм для защиты от спама и некорректных данных, а также использование PostgreSQL в качестве базы данных и Poetry для управления зависимостями.

## Основные возможности

*   **Приложение "Каталог" (`catalog`)**:
    *   Управление продуктами и категориями товаров.
    *   Использование классовых представлений (Class-Based Views, CBV) для всех пользовательских интерфейсов.
    *   Страницы для отображения товаров, контактной информации и формы для добавления новых продуктов.
*   **Приложение "Блог" (`blog`)**:
    *   Полный цикл управления статьями (Create, Read, Update, Delete - CRUD).
    *   Автоматическая генерация и транслитерация SEO-дружественных URL-идентификаторов (slug).
    *   Учет количества просмотров статей.
    *   Отображение только опубликованных статей в общем списке.
*   **Административная панель Django**:
    *   Централизованное управление всеми данными проекта (товары, категории, записи блога, пользователи)
    *   Полная локализация интерфейса на русский язык.
*   **База данных**:
    *   Использование PostgreSQL для хранения данных проекта.
*   **Управление зависимостями**:
    *   Применяется Poetry для изоляции окружения и управления пакетами.

## Начало работы

Для локального развертывания и запуска проекта выполните следующие шаги.

### 1. Клонирование репозитория

```bash
git clone <URL вашего репозитория>
cd django_projects_new
```

### 2. Установка зависимостей

Установите все необходимые зависимости с помощью Poetry:
```bash
poetry install
```

### 3. Настройка переменных окружения

Создайте файл `.env` в корневой директории проекта на основе `.env.template`. Заполните его необходимыми данными для подключения к базе данных PostgreSQL и секретным ключом Django.

Пример `.env`:
```ini
DB_ENGINE=django.db.backends.postgresql
DB_NAME=django_catalog_db
DB_USER=your_db_user
DB_PASSWORD=your_db_password
DB_HOST=localhost
DB_PORT=5432

SECRET_KEY=your_very_secret_django_key_here
# Для необязательной отправки email (если настроено в settings.py)
# EMAIL_BACKEND = 'django.core.mail.backends.smtp.EmailBackend'
# EMAIL_HOST = 'smtp.gmail.com'
# EMAIL_PORT = 587
# EMAIL_USE_TLS = True
# EMAIL_HOST_USER = 'your_email@gmail.com'
# EMAIL_HOST_PASSWORD = 'your_app_password' # Используйте пароль приложения
```

### 4. Применение миграций

Создайте структуру базы данных:
```bash
python manage.py migrate
```

### 5. Заполнение базы данных начальными данными

Загрузите начальные категории и продукты из фикстур:
```bash
python manage.py fill
```

### 6. Создание суперпользователя

Для доступа к административной панели создайте суперпользователя:
```bash
python manage.py createsuperuser
# Если у вас есть кастомная команда `createsu` с предустановленными данными:
# python manage.py createsu
```

### 6. Сбор статических файлов

Соберите все статические файлы проекта:
```bash
python manage.py collectstatic --noinput
```

### 7. Запуск сервера разработки

Запустите локальный сервер разработки Django:
```bash
python manage.py runserver
```

Теперь проект доступен по адресу `http://127.0.0.1:8000/`.

**Ключевые адреса:**
*   **Главная страница:** `http://127.0.0.1:8000/`
*   **Административная панель:** `http://127.0.0.1:8000/admin/`
*   **Список статей блога:** `http://127.0.0.1:8000/blogs/`
*   **Страница контактов:** `http://127.0.0.1:8000/contacts/`
