# Skystore

Это учебный проект в рамках курса по веб-разработке на Django. Проект представляет собой основу для создания интернет-магазина.

## Описание

Проект "Skystore" представляет собой базовую платформу для интернет-магазина, разработанную с использованием Django. Он включает в себя основные функции для управления товарами и категориями, а также демонстрацию этих данных.

## Основные возможности

*   **Управление товарами и категориями**: Модели `Product` и `Category` для структурированного хранения данных о товарах.
*   **Админ-панель**: Полностью настроенная административная панель Django для удобного создания, редактирования и удаления товаров, категорий и контактной информации.
*   **Кастомная команда `fill`**: Утилита командной строки `python manage.py fill` для автоматического заполнения базы данных тестовыми данными (категориями и продуктами).
*   **Главная страница**: Отображение последних 5 созданных товаров на главной странице проекта.
*   **Страница контактов**: Страница с контактной информацией, извлекаемой из базы данных, и формой обратной связи.
*   **Подключение к PostgreSQL**: Настройка проекта для работы с базой данных PostgreSQL.

## Установка и запуск проекта

Следуйте этим шагам, чтобы развернуть и запустить проект локально.

### 1. Клонирование репозитория

```bash
git clone <URL вашего репозитория>
cd django_projects_new
```

### 2. Установка зависимостей

Проект использует Poetry для управления зависимостями.
```bash
poetry install
```

### 3. Настройка базы данных PostgreSQL

*   Создайте новую базу данных PostgreSQL (например, `skystore_db`).
*   Создайте файл `.env` в корне проекта на основе `.env.template`.
*   Заполните `.env` своими учетными данными для подключения к PostgreSQL:

    ```
    # PostgreSQL database settings
    DB_ENGINE=django.db.backends.postgresql
    DB_NAME=skystore_db
    DB_USER=your_db_user
    DB_PASSWORD=your_db_password
    DB_HOST=localhost
    DB_PORT=5432

    # Django secret key
    SECRET_KEY=your_secret_key_for_production (используйте длинную случайную строку)
    ```

### 4. Применение миграций

```bash
python manage.py migrate
```

### 5. Создание суперпользователя

Для доступа к админ-панели используйте нашу кастомную команду, которая создаст суперпользователя на основе переменных, указанных в вашем файле `.env` (или используя значения по умолчанию):
```bash
python manage.py createsu
```
Убедитесь, что в вашем файле `.env` настроены переменные `SU_USERNAME`, `SU_EMAIL`, `SU_PASSWORD`. Если они не указаны, будут использоваться значения по умолчанию: `admin`, `admin@example.com`, `admin` соответственно.

### 6. Заполнение базы тестовыми данными

Используйте кастомную команду для заполнения базы данных:
```bash
python manage.py fill
```

### 7. Запуск сервера разработки

```bash
python manage.py runserver
```

Теперь вы можете открыть проект в браузере: `http://127.0.0.1:8000/`

### 8. Добавление контактных данных в админ-панели

После запуска сервера, зайдите в админ-панель (`http://127.0.0.1:8000/admin/`), авторизуйтесь и добавьте контактную информацию в разделе "Контакты", чтобы она отображалась на странице контактов.
